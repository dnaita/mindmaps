Getting Started with the Node.js API

Built on top of the REST API, the MarkLogic Node.js API provides a native programming interface for JavaScript developers to use for creating Node.js applications that use a MarkLogic database.

REST API Instance

In order to be able to use the Node.js API to communicate with a MarkLogic database, a REST API instance needs to exist for the database.

Using the Node.js API

Available in the public npm repository (www.npmjs.com/package/marklogic), the MarkLogic Node.js API package can be downloaded to a Node.js project through the simple command of npm install marklogic.

To use the downloaded package, the require function can be used. All of the MarkLogic Node.js API functionality then becomes available to the Node.js project.

var marklogic = require('marklogic');

A connection can then be created, that uses a REST API instance to connect to a MarkLogic database:

var connection = {
  host: 'localhost',
  port: 8060,
  user: 'admin',
  password: 'admin'
};

Documents can then be retrieved from the database:

var db = marklogic.createDatabaseClient(connection);

db.documents.read('/patents/airless_diving.json', '/patents/waterski_drone.json')

   .result(function(documents) {
      documents.forEach(function(document) {...})
   });

Reference Instance

The Samplestack reference instance (available at github.com/marklogic/marklogic-samplestack) provides a 3-tier, enterprise reference architecture for Node.js developers to use for creating their own MarkLogic applications.

Implemented using Angular, Node.js, Express, Gulp, and MarkLogic, Samplestack includes enterprise-grade features such as multi-statement transactions, role-based security, and LDAP integration.
