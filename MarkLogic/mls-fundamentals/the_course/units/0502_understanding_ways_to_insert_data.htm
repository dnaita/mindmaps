Understanding Ways to Insert Data

There are many ways to insert documents into a MarkLogic database. Ways include:

  • MarkLogic Content Pump
  • REST API
  • Java API
  • Node.js API
  • XCC
  • XQuery functions
  • JavaScript functions
  • WebDAV
  • MarkLogic Connector for Hadoop
  • Content Processing Framework

 

MarkLogic Content Pump is a command-line tool for bulk loading billions of documents into a database.

The REST API provides a programming language-agnostic way to write documents to a database.

The Java API provides Java classes for developers to use to write documents to a database from their Java code.

The Node.js API provides Node.js classes for developers to use to write documents to a database from their Node.js code.

XCC is a low-level interface that allows Java or .NET applications to write documents to a MarkLogic database.

XQuery functions can be used to write documents to a database from Query Console or from an XQuery application.

JavaScript functions can be used to write documents to a database from Query Console or from a JavaScript application.

WebDAV can be used to drag and drop documents into a MarkLogic database.

MarkLogic Connector for Hadoop is a set of Java classes that enables using a MarkLogic database as a Hadoop MapReduce input source and/or output destination.

The Content Processing Framework is a pipeline framework for making changes to documents as they are being loaded into a database, such as enriching the data, or transforming it from an MS Office or PDF format to XML.
