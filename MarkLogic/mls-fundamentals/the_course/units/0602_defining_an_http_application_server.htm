Defining an HTTP Application Server

An HTTP Application Server can be defined through the Admin Interface, by using the Create HTTP tab.


The database field is for specifying which database the application server is for.

The modules field is for specifying whether the application files will live on the file system or in a database.

The root field is for specifying the exact location of the application files.

The port field is for specifying which port the application will be available on.

The server name field is for giving the application server a name.

Once the fields have been filled out and the ok button clicked, the application server can be used.
