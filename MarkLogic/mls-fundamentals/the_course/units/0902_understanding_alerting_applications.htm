Understanding Alerting Applications

MarkLogic offers the capability to have a developer create an application that sends an alert whenever a document that meets certain criteria is inserted (or updated) in a database.

For example, a match-making application, like a job posting site (or a dating site or ride-sharing site), might use alerting to notify users whenever a new job (or dater or ride-sharer) is inserted that meets their criteria.

Alerting is accomplished by having a query stored in MarkLogic (stored in a database, serialized), setting the fast reverse searches index to true, setting up a trigger, and specifying an action.

Alerting applications are commonly created in conjunction with the Content Processing Framework, which has a pre-built pipeline for alerting.

Both documentation (docs.marklogic.com/guide/search-dev/alerts) and training (Developing MarkLogic Applications ll) are available on creating alerting applications in MarkLogic.
