Getting Started with the Java API

Built on top of the REST API, the Java API provides a native interface for Java developers to create applications that use a MarkLogic database.

REST API Instance

In order to be able to use the Java API to communicate with a MarkLogic database, a REST API instance needs to exist for the database.

Getting the Java API Files

The Java API files can be acquired from the MarkLogic Community site (developer.marklogic.com/products/java) or from GitHub (github.com/marklogic/java-client-api).

Reference Instance

The Samplestack reference instance (available at github.com/marklogic/marklogic-samplestack) provides a 3-tier, enterprise reference architecture for Java developers to use for creating their own MarkLogic applications.

Implemented using Angular, Gradle, Spring, Java, and MarkLogic, Samplestack includes enterprise-grade features such as multi-statement transactions, role-based security, and LDAP integration.

Resources

The Java Application Developer's Guide (docs.marklogic.com/guide/java.pdf) is a great resource for Java developers looking to use the Java API for communicating with a MarkLogic database.

In addition, Java API tutorials exist on the MarkLogic Community site (developer.marklogic.com/try/java/index) and (developer.marklogic.com/learn/java). Or, to learn about the full-length, instructor-led Developing MarkLogic Applications I: Java course, see mlu.marklogic.com/registration.
