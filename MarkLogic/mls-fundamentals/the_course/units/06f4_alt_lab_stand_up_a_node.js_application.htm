Alternate Lab Steps: Stand up a Node.js Application (optional)

In this lab exercise, you stand up a Node.js star-wars application.

 

  1. In the mls-fundamentals/unit06 folder, open the star-wars.pdf file to see a screenshot of the application that we will be standing up.
  2. In Query Console, check that you still have the 51 documents in your star-wars database.
  3. Create a REST API instance that communicates with the star-wars database through port 8071.
  4. On the desktop of the virtual machine, copy the star-wars folder from the mls-fundamentals/unit06 folder to the mls-projectsfolder.
  5. In the mls-projects/star-wars folder, open package.json in a text editor to see marklogic listed in the dependencies.
  6. In a command prompt window, navigate to the star-wars project folder and run npm install, to install the dependencies listed in the package.json file.

    Note: Requires Internet access.
  7. In the project files, see that there is now a folder called node_modules and that, in it, there is a folder named marklogic.
  8. Back in the mls-projects/star-wars folder, open the app.js file in a text editor.
  9. Notice the port of 8072 for the Node.js application.
  10. Notice the require('./routes') lines:

  11. In the mls-projects/star-wars folder, open the routes.js file in a text editor.
  12. Notice the require('marklogic') line, the require('./settings') line, as well as additional variables to implement the application's functionality:

  13. Notice a reference to a collection named character (line 48).
  14. In Query Console, see that none of the star-wars documents are currently in collections:


  15. Open unit06/labb.txt:

  16. Run the JavaScript code in Query Console to assign the documents to collections.
  17. Check that the documents are now in collections named character and image:

  18. In the mls-projects/star-wars folder, open the settings.js file in a text editor.
  19. See that this file contains settings for the application to connect to the star-wars database through the REST API instance at port 8071:

  20. If desired, continue perusing the application folders and files.
  21. In a command prompt window, in the mls-projects/star-wars directory, enter bower install, to install application dependencies.

    Note: Requires Internet access.
  22. In the command prompt window, in the star-wars project directory, start the star-wars application.
  23. In a browser, enter http://localhost:8072 to access the application:


  24. In the menu bar, click Characters.
