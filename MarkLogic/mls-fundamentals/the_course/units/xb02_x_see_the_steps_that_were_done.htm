Exercise: See the Steps That Were Done

In this exercise, you view the steps that were done to create the CentOS virtual machine for this course. Realize that, if you go to reproduce these steps, some of the versions and URLs may have moved on.

Note: It is not necessary to build the virtual machine yourself. You can download it from MarkLogic University.

 

  1. Downloaded and installed VirtualBox (VirtualBox-4.3.28-100309-Win.exe).
  2. Downloaded CentOS ISO (CentOS-6.7-x86_64-bin-DVD1.iso).

To create the virtual machine:

  1. In VirtualBox, clicked the New button.
  2. For Name, entered centos_64.
  3. For Type, left Linux selected.
  4. For version, left Red Hat (64 bit) selected.
  5. Clicked the Next button.
  6. For Memory size, entered 1536 (1.5 GB), and clicked the Next button.
  7. Left Create a virtual hard drive now selected, and clicked the Create button.
  8. Left VDI (VirtualBox Disk Image) selected, and clicked the Next button.
  9. Left Dynamically allocated selected, and clicked the Next button.
  10. For file location, left the default location of centos_64 (C:\Users\rstryker\VirtualBox VMs\centos_64).
  11. For size, entered 20 GB, and clicked the Create button.

To use the CentOS ISO image for the virtual machine:

  1. With the virtual machine selected on the left, clicked the Storage heading.
  2. In the Storage Tree section, under Controller: IDE, selected Empty:

  3. In the Attributes section, for CD/DVD Drive, clicked the CD icon to the right of the drop-down list box, to select Choose a virtual CD/DVD disk file....
  4. Navigated to select the CentOS ISO downloaded in Step 2.
  5. Clicked the Open button and clicked the OK button.

To initialize the virtual machine:

  1. Clicked the Start button, to start the virtual machine.
  2. In the boot screen, left Install or upgrade an existing system selected and pressed the Enter key.
  3. Watched the installation steps begin.
  4. When prompted, selected Skip, to skip testing the media (i.e. right arrow, then Enter).

    Note: Can press the right Control key at any time, to use mouse outside of the virtual machine.
  5. Pressed the Next button, to proceed with the installation.
  6. Left English (English) selected, and clicked the Next button.
  7. Left U.S. English selected, and clicked the Next button.
  8. Left Basic Storage Devices selected, and clicked the Next button.

    Note: Can close the information panels at top, if they are blocking the view. (Can use the right Control key to get outside.)
  9. Selected the Yes, discard any data button.
  10. In the Hostname window, clicked the Configure Network button (in the lower-left corner).
  11. Selected System eth0 and pressed the Edit... button.
  12. Selected the checkbox for Connect automatically.
  13. Clicked the Apply... button.
  14. Clicked the Close button.
  15. For Hostname, entered centos-vm.
  16. Clicked the Next button.
  17. Selected Europe/London and clicked the Next button.
  18. For Root Password, entered centos.
  19. For Confirm, entered centos.
  20. Clicked the Next button.
  21. In the Weak Password dialog box, clicked the Use Anyway button.
  22. Selected the Use All Space radio button, and clicked the Next button.
  23. Clicked the Write changes to disk button.
  24. Selected Minimal Desktop.
  25. Clicked the Next button.
  26. Watched the installation complete.
  27. Clicked the Reboot button.
  28. Waited for the virtual machine to reboot.
  29. Clicked the Forward button.
  30. Clicked the Forward button again.
  31. In the Create User screen, for Username, entered cent.
  32. For Full Name, entered cent.
  33. For Password and Confirm Password, entered centos.
  34. Clicked the Forward button.
  35. In the weak password dialog box, clicked the Yes button.
  36. In the Date and Time tab, entered the current date and time, and clicked the Forward button.
  37. In the Kdump insufficient memory dialog box, clicked the OK button.
  38. Clicked the Finish button.
  39. Waited for the virtual machine to reboot.
  40. Clicked the cent button.
  41. For Password, entered centos and clicked the Log In button.

To open ports:

  1. Selected Applications - System Tools - Terminal.
  2. Entered su -.
  3. For password, entered centos.
  4. Entered /etc/init.d/iptables save.
  5. Entered /etc/init.d/iptables stop.
  6. Entered chkconfig iptables off.

To turn off transparent huge pages, set the I/O scheduler to deadline, and set huge pages to 3/8 system memory (virtual machine has 1.5 MB RAM):

  1. Entered vim /boot/grub/grub.conf.
  2. Pressed the i key to be able to edit the file.
  3. At the end of the kernel line, added:

    transparent_hugepage=never elevator=deadline
  4. Pressed the Esc key.
  5. Typed :wq and then pressed the Enter key to exit the file.
  6. Entered vim /etc/sysctl.conf.
  7. Pressed the i key to be able to edit the file.
  8. Scrolled to the bottom of the file.
  9. Added vm.nr_hugepages=281.
  10. Pressed the Esc key.
  11. Typed :wq and then pressed the Enter key to exit the file.

To update the CentOS packages, and install packages that will be needed for installing MarkLogic:

  1. Entered yum update (and then y when prompted, twice).
  2. Entered yum install gcc (and then y when prompted).
  3. Entered yum install kernel-devel (and then y when prompted).
  4. Entered halt # reboot (and then logged back in to root by, in VirtualBox, clicking the Start button for the virtual machine, waiting for it to start up, clicking cent, entering Password of centos, selecting Applications - System Tools - Terminal, entering su -, and entering centos).
  5. Entered yum groupinstall 'Development Tools' (and then y when prompted).
  6. Entered yum install gdb glibc glibc.i686 lsb (and then y when prompted).
  7. Entered exit to log out of root.
  8. Entered exit again, to close the terminal window.

To install Guest Additions (to be able to copy and paste text between the virtual machine and the host computer, and easily use the mouse between the virtual machine and the host computer):

  1. Selected Devices - Insert Guest Additions CD image....
  2. In the dialog box, left Open Autorun Prompt selected and clicked the OK button.
  3. Clicked the Run button.
  4. Entered the root password of centos and clicked the Authenticate button.
  5. Waited for it to install.
  6. Pressed Enter at the end of the script.
  7. Selected Devices - Shared Clipboard - Bidirectional.
  8. Selected Devices - Drag'n'Drop - Bidirectional.
  9. Restarted the virtual machine by selecting System - Shut Down - Restart button.
  10. Clicked cent and entered the password of centos, to log in to the restarted virtual machine.
  11. Right-clicked the guest additions CD on the desktop, to then select Eject, to eject the CD.

To install gedit, to be able to edit text files (for Unit 4 onward):

  1. Selected Applications - System Tools - Terminal.
  2. Entered su -.
  3. For password, entered centos.
  4. Entered yum install gedit (and then y when prompted).

To install evince, to be able to view PDF files (for Unit 1 lab and Unit 6 lab):

  1. Entered yum install evince (and then y when prompted).

To check that curl is installed (for Unit 4 onward):

  1. Entered curl --version.

To download MarkLogic Content Pump (for Unit 5):

  1. In the virtual machine, launched Firefox, and entered http://developer.marklogic.com/code.
  2. Clicked the + icon to the left of the Loading and processing category, to expand it.
  3. Clicked the MarkLogic Content Pump link.
  4. Clicked the Release 1.3-3 binaries zip package link.
  5. Selected the Save File radio button, and clicked the OK button.
  6. Watched the progress bar, to see that the .zip file downloaded.
  7. Opened a new terminal window (Applications - System Tools - Terminal), and, as cent user, entered cd Downloads.
  8. Entered ls to see the mlcp-1.3-3-bin.zip file listed.
  9. Entered unzip mlcp-1.3-3-bin.zip.
  10. On the desktop, double-clicked cent's Home folder.
  11. Double-clicked the Downloads folder.
  12. Dragged the expanded mlcp-1.3-3-bin folder to the desktop.

 

For using the MarkLogic Java API (in Unit 6), downloaded and installed JDK, Maven, Tomcat, and Eclipse:

  1. In a terminal window, entered su -, and then the password of centos, to be root.
  2. Entered yum remove java-1.7.0-openjdk, to remove OpenJDK (and then y when prompted).
  3. In Firefox, to download the JDK, went to http://www.oracle.com/technetwork/java/javase/downloads/index.html.
  4. Selected the Download button for JDK 8:


  5. Selected the Accept License Agreement radio button.
  6. Selected the Linux x64 rpm download:


  7. Selected the Save File radio button, and clicked the OK button.
  8. Waited for the .rpm file to download (by watching the orange progress bar in the upper-right corner of Firefox).
  9. In a terminal window, as root, entered cd /home/cent/Downloads.
  10. Entered ls to see the jdk-8u65-linux-x64.rpm file listed.
  11. Entered rpm -ivh jdk-8u65-linux-x64.rpm.
  12. Entered java -version, to see that it installed:


  13. Entered javac -version, to see that javac is set up:


  14. Entered the following commands, to set up global environment variables:

    export JAVA_HOME=/usr/java/jdk1.8.0_65/
    export PATH=$PATH:$JAVA_HOME
  15. Entered echo $JAVA_HOME and echo $PATH to see that they are set up:


  16. To create a permanent file (so that the variables can still exist after rebooting the virtual machine), entered vim /etc/profile.d/java.sh.
  17. Pressed the i key to be able to edit the file.
  18. In the file, entered:

    #!/bin/bash
    JAVA_HOME=/usr/java/jdk1.8.0_65/
    PATH=$JAVA_HOME/bin:$PATH
    export PATH JAVA_HOME
    export CLASSPATH=.
  19. Pressed the Esc key.
  20. Typed :wq and then pressed the Enter key to exit the file.
  21. On the command line, entered chmod +x /etc/profile.d/java.sh to make the file executable.
  22. Entered source /etc/profile.d/java.sh to set the environment variables permanently.


  23. To install Maven (in Unit 6, used to install the MarkLogic Java API files in an Eclipse project), went to http://maven.apache.org/download.cgi in Firefox.
  24. Selected the binary zip file for the current stable version of Maven:


  25. Left the Save File radio button selected and clicked the OK button.
  26. Waited for the .zip file to download (by watching the orange progress bar).
  27. In a terminal window, as root, entered cd /home/cent/Downloads.
  28. Entered ls to see the apache-maven-3.3.3-bin.zip file listed.
  29. Entered unzip apache-maven-3.3.3-bin.zip.
  30. Entered ls to see the apache-maven-3.3.3 directory listed.
  31. Entered mv apache-maven-3.3.3 /opt to move the expanded Maven files to the /opt directory.
  32. Entered export PATH=/opt/apache-maven-3.3.3/bin:$PATH to add Maven's bin directory to the PATH variable.
  33. Entered mvn -version to see that Maven is set up:


  34. To add Maven to the PATH variable permanently (so that it is still in the PATH variable even after rebooting the virtual machine), entered vim /etc/profile.d/java.sh.
  35. Pressed the i key to be able to edit the file.
  36. In the file, added (the fourth line):

    #!/bin/bash
    JAVA_HOME=/usr/java/jdk1.8.0_65/
    PATH=$JAVA_HOME/bin:$PATH
    PATH=/opt/apache-maven-3.3.3/bin:$PATH
    export PATH JAVA_HOME
    export CLASSPATH=.
  37. Pressed the Esc key.
  38. Typed :wq and then pressed the Enter key to exit the file.


  39. To download Tomcat (in Unit 6, used for running a Java servlet), went to http://tomcat.apache.org/download-80.cgi.
  40. In Binary Distributions, under Core, selected zip:

  41. Left the Save File radio button selected and clicked the OK button.
  42. Waited for the .zip file to download.
  43. In a terminal window, as root, entered cd /home/cent/Downloads.
  44. Entered ls to see the apache-tomcat-8.0.28.zip file listed.
  45. Entered unzip apache-tomcat-8.0.28.zip.
  46. Entered ls to see the apache-tomcat-8.0.28 directory listed.
  47. Entered mv apache-tomcat-8.0.28 /opt to move the Tomcat directory to the /opt directory.
  48. Entered ls /opt to see the tomcat directory.


  49. In Firefox, to download Eclipse, went to http://eclipse.org/downloads.

  51. Clicked the DOWNLOAD button, on the left, for the eclipse-jee-mars-1-linux-gtk-x86_64.tar.gz file.
  52. Left the Save File radio button selected, and clicked OK.
  53. Waited for the .zip file to download (by watching the orange progress bar).

  56. Entered tar xvf eclipse-jee-mars-1-linux-gtk-x86_64.tar.gz.

To shrink the size of the virtual machine:

  1. Opened a terminal window, entered su -, and then the password of centos, to be root.
  2. Entered cd /home/cent/Downloads.
  3. Entered ls to see the files in the Downloads folder.
  4. Entered rm -rf *.
  5. Entered ls to see that the files in the Downloads folder are gone.
  6. Entered dd if=/dev/zero of=zerofillfile bs=1M.
  7. Waited for the command to finish running.
  8. Entered rm zerofillfile (and then y when prompted).
  9. Entered exit, to exit from being root.
  10. Closed any open windows.
  11. Selected System - Shut Down..., and then clicked the Shut Down button, to shut down the virtual machine.
  12. In Windows (i.e. the host computer), opened a terminal window.
  13. Entered cd "c:\Program Files\Oracle\VirtualBox".
  14. Entered VBoxManage modifyhd --compact "C:\Users\rstryker\VirtualBox VMs\centos_64\centos_64.vdi".
  15. Waited for it to finish (about 2 minutes).
  16. In VirtualBox, with the virtual machine selected, clicked Start, and logged in by clicking cent and entering the password of centos.

To create a clone of the centos_64 virtual machine:

  1. Closed all open windows.
  2. Shut down the virtual machine by selecting System - Shut Down..., and clicking the Shut Down button.
  3. In VirtualBox, with the virtual machine selected, selected Machine - Clone.
  4. For new machine name, entered centos_64_fundamentals.
  5. Selected the checkbox for Reinitialize the MAC address of all network cards.
  6. Clicked the Next button.
  7. Left Full clone selected.
  8. Clicked the Clone button.
  9. Waited for the new virtual machine to be generated.

To generate an .ova file for distribution:

  1. Selected File - Export Appliance.
  2. With the cloned virtual machine highlighted (centos_64_fundamentals), clicked the Next button.
  3. For File, specified a location and a file name of centos_64_fundamentals.ova (e.g. C:\Users\rstryker\Desktop\centos_64_fundamentals.ova).
  4. Clicked the Next button.
  5. Clicked the Export button.
  6. Waited for the .ova file to be generated.
  7. Uploaded the .ova file for distribution.
