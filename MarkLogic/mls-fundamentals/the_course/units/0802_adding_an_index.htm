Adding an Index

The Admin Interface can be used to add an index to a MarkLogic database.

If the index to be added is one of the indexes available through the true and false radio buttons, the true button can be selected and then the ok button pressed.

Adding an index does not require a restart or system delay. As an index is being set up, MarkLogic re-indexes the database in the background. While it is re-indexing in the background, any queries to the database continue to be served. Once the index is available, any new queries will take advantage of the index if useful to the query.

Element Range Index

An element range index can be added that contains the values of an element from the documents in the database. For example, an element range index could be set up for all of the name values in the documents in the star-wars database.

If providing a way for a user to sort the values of a property within documents in a MarkLogic database, a range index can greatly improve the performance of the sorting (e.g. if there are millions of documents in the database).
