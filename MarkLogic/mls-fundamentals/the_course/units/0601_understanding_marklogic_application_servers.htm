Understanding MarkLogic Application Servers

MarkLogic application servers provide an interface for communicating with MarkLogic databases, as well as, in the case of a MarkLogic HTTP application server, the option of referencing application files for a web application.

Four types of application servers can be created for a database: HTTP (for a web application and/or REST API interface), WebDAV (for dragging and dropping documents into a database), XDBC (for low-level Java and .NET connections through XCC), and ODBC (to enable business intelligence tools such as Tableau or Cognos to connect to a MarkLogic database).

All of the application server types are available through tabs in the Admin Interface, under Configure - Groups - Default - App Servers.


Modules Database

A modules database is a normal database, but used for storing application files rather than data documents.
