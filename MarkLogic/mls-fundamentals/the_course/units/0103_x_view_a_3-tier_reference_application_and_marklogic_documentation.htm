Exercise: View a 3-Tier Reference Application and MarkLogic Documentation

In this walkthrough exercise, you view a 3-tier application and then access the MarkLogic documentation to see where you could learn more about how MarkLogic works.

 

  1. In a browser window, navigate to docs.marklogic.com/guide/ref-arch.pdf.
  2. Skim the table of contents:


  3. Skim through the guide, as desired.
  4. Navigate to github.com/marklogic/marklogic-samplestack, to access the code for Samplestack, a 3-tier reference application that Java or Node.js developers can use as an example for creating an enterprise application with MarkLogic.
  5. In a browser, navigate to docs.marklogic.com.
  6. See that you are on the landing page for the MarkLogic documentation:


  7. Scroll down to see the many guides under the various sections (e.g. Getting Started Guides, Developer's Guides, Administrator's Guides, Other Reference Guides).
  8. In the Developer's Guides section, notice the REST Application Developer's Guide, the Java Application Developer's Guide, the Node.js Application Developer's Guide, and the Application Developer's Guide.
  9. If desired, click the Application Developer's Guide link, to see that chapter 3 (Understanding Transactions in MarkLogic Server) details how transactions work in MarkLogic.
  10. Click the browser's Back button, to return to the list of guides.
  11. In the Other Reference Guides section, select the PDF icon for the Scalability, Availability, and Failover Guide:


  12. Scroll down to view the table of contents, noting that this guide provides detail on scalability and high availability in MarkLogic.
  13. Click the heading for section 3. Scalability Considerations in MarkLogic Server.
  14. Skim through the topic.
  15. Continue perusing the guide, as desired.
