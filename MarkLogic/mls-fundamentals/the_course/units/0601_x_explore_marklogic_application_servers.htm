Exercise: Explore MarkLogic Application Servers

In this walkthrough exercise, you explore the existing application servers, already set up.

 

  1. In the Admin Interface (http://localhost:8001), select Configure - Groups - Default - App Servers.
  2. Notice the types of application servers available, in the tabs:


  3. In the Summary tab, notice the application server type, database, and port number for each of the application servers already set up:


  4. Notice the application server for port 8001.
  5. Click on Admin, to see the settings for the application server.
  6. Realize that it is serving the application that we are using (i.e. the Admin Interface).
  7. Notice that the modules field is set to (file system).
  8. Realize that application files could be stored in a database instead of on the file system.
  9. Notice that the root path is Admin/.
  10. If desired, in a command prompt window, enter the following command to see the files for this application:

    ls /opt/MarkLogic/Admin

In Query Console, view references to application servers and modules databases:

  1. In a browser window, go to Query Console.
  2. Look in the Content Source list:


  3. Realize that the parentheses to the right of a database are simply for showing any application servers attached to the database.
  4. Notice that the patents database has an application server connected to it (created along with the REST API instance in Unit 4).
  5. In the Admin Interface, select Configure, to see the list of application servers in the Summary tab.
  6. In the App Servers column, click 8060-patents, to see the settings for it.
  7. Notice that the root field has a value of /, and the modules field is set to the database named 8060-patents-modules.
  8. Realize that this HTTP application server, and the modules database, were created in Unit 4, when we created the REST API instance for the patents database.
  9. In Query Console, select the 8060-patents-modules database.
  10. Click the Explore button, to see the contents of this database:


  11. Realize that these REST API supporting documents were created when the REST API instance was created.
