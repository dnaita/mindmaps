Exercise: Discuss Indexes

In this walkthrough exercise, you discuss the indexes for the Top Songs application. Then, you view the default index settings for a database.

 

  1. In a browser, navigate to mlu.marklogic.com/top-songs.
  2. With facets requiring an index, realize that the artist facet is using an index of the values in the artist element of the song documents, that the decade facet is using an index of values in the week elements in the song documents, and that the genre element is using an index of the values in the genre elements in the song documents.
  3. In looking at the sort menu, realize that indexes (e.g. values in the week element of the song documents for sort:newest and sort:oldest, values in the artist element of the song documents for sort:artist, and values in the title element of the song documents for sort:title) make it so the sorts perform quickly.
  4. In looking at the birthday search form field, realize that an index of the values in the week element of the song documents enables the search to happen quickly.
  5. Click the advanced search link.
  6. Notice the artist/writer/producer form field.
  7. Realize that an index with the values in the artist, writer, and producer elements from the song documents enables this search to happen quickly.
  8. View the genre drop-down list.
  9. Realize that an index with the values from the genre elements of the song documents makes this list possible.
  10. In the Admin Interface, select Configure - Databases - patents.
  11. Scroll down to view the true and false radio buttons for a variety of indexes available for the database.
  12. Back towards the top, in the left navigation, notice additional types of indexes that you could set up for the database (e.g. Element Range Indexes, Fields, Geospatial Indexes).
