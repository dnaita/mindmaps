Understanding the Search Programming Interfaces

To allow an application to search the documents in a MarkLogic database, there are a number of programming interfaces available.

cts Functions

At the lowest level, are the cts (core text search) functions. The cts functions have always existed in MarkLogic and serve as the basis for the higher-level APIs.

Search API

The Search API, built upon the cts functions, is a higher-level API that includes best practices while also providing useful application features such as highlighting, snippets, pagination, and faceting.

REST API search Service

Built on the Search API, the REST API search service provides a way to query a database through a REST API instance. This is quite useful to developers who are using MarkLogic for their database, but writing their application code in a language other than XQuery, JavaScript, or Java.

Java API and Node.js API

Built on the REST API, the Java and Node.js APIs provide MarkLogic search capabilities for Java and Node.js developers in their native language.
