    • + - cloud service models-3

      • + - Infrastructure as a Service (IaaS),
        • + - def:
          •  is about providing compute and storage resources as a service.

            As per NIST
            "The capability provided to the consumer is to provision processing, storage, networks, and other fundamental computing resources where the consumer is able to deploy and run arbitrary software, which can include operating systems and applications. The consumer does not manage or control the underlying cloud infrastructure but has control over operating systems, storage, deployed applications, and possibly limited control of select networking components (e.g., host firewalls)."

        • Well-known IaaS platforms include Amazon EC2, Rackspace, and Rightscale.
      • + - Platform as a Service (PaaS)

        • + - def:

          •  provide a system stack or platform for application deployment as a service.

            The capability provided to the consumer is to deploy onto the cloud infrastructure consumer-created or acquired applications created using programming languages and tools supported by the provider. The consumer does not manage or control the underlying cloud infrastructure including network, servers, operating systems, or storage, but has control over the deployed applications and possibly application hosting environment configurations.

           Windows Azure, Google App Engine, and Hadoop are some well-known PaaS platforms

      • + - Software as a Service (SaaS)

        • def:

          • about providing the complete application as a service.

            The capability provided to the consumer is to use the provider's applications running on a cloud infrastructure. The applications are accessible from various client devices through a thin client interface such as a web browser (e.g., web-based email). The consumer does not manage or control the underlying cloud infrastructure including network, servers, operating systems, storage, or even individual application capabilities, with the possible exception of limited user-specific application configuration settings. Users who log in to the SaaS service can both use the application as well as configure the application for their use.

           Prominent SaaS applications include Salesforce.com for CRM, Google Docs for document sharing, and web email systems like Gmail, Hotmail, and Yahoo! Mail.

          EC2 Storage Resources

          • + -

             block storage resources.

            • + - Elastic Block Service (EBS)
              •  provides a block storage service for EC2.  It is possible to request an EBS disk volume of a particular size and attach this volume to one or multiple EC2 instances using the instance ID returned during the time the volume is created.  the EBS volume has an existence independent of any EC2 instance, which is critical to have persistence of data
              + - Instance Storage:
              •  EC2 instance has local storage that can be configured as a part of the compute resource  This instance storage is ephemeral it exists only as long as the EC2 instance exists,  and cannot be attached to any other EC2 instance.  if the EC2 instance is terminated, the instance storage ceases to exist.
              S3-backed instances vs. EBS-backed instances:
              •  In an S3-backed instance, the root AMI is stored in S3, which is file storage. Therefore, it must be copied to the root device in the EC2 instance before the EC2 instance can be booted.  since instance storage is not persistent, any modifications made to the AMI of an S3-backed instance (such as patching the OS or installing additional software) will not be persistent beyond the lifetime of the instance.

          EC2 Networking Resources

          • EC2 offers both a public address as well as a private address also offers DNS services for managing DNS names associated with these IP addressees. 

            The Virtual Private Cloud can be used to provide secure communication between an Intranet and the EC2 network. 

            . One can also create a complete logical sub network and expose it to public (a DMZ) with its own firewall rules. Elastic IP addresses which are independent of any instance, and this feature can be used to support failover of servers.  + - Instance addresses:
            •  Each EC2 instance has two IP addresses associated with it  + -  public IP address
              • public IP address and DNS name can be used for communication outside the Amazon cloud
              + - private IP address
              •  private IP address and DNS name can be resolved only within the EC2 cloud.
            + - Elastic IP addresses: 
            •  are independent of any instance, but are associated with a particular Amazon EC2 account  can be dynamically assigned to any instance (in which case, the public IP address is de-assigned).  they are useful for implementing failover.  Elastic IP addresses are not automatically allocated; they have to be generated when needed.
            + - Route 53:

            • is a DNS server that can be used to associate an Elastic IP address or public IP address with a name of the form www.myenterprise.com.

              This is not possible by default, since the EC2 instances are inside the amazon.com domain. 

          + - Security Groups
          •  to define network security policies that restrict the ports through which any machine can be accessed, or the IP addresses that can access a server.   Each security group is a collection of network security policies.  The default security group when creating an EC2 instance allows the instance to connect to any outside IP address but disallows incoming connections.
          + -

          Virtual Private Cloud:

          • Its for enterprises that desire more control over their networking configuration.

            + - advanced networking features offered by VPC include:

              • the ability to allocate both public and private IP addresses to instances from any address range

              • the ability to divide the addresses into subnets and control the routing between subnets

              • the ability to connect the EC2 network with an Intranet using a VPN tunnel. Details of VPC are beyond the scope of this book and can be found in Amazon Virtual Private Cloud[11].
